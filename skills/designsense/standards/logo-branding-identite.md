# Logo, branding & identité — standards UI/UX

> Règles pour un agent qui construit du front. Agnostique de charte. Appliquer par défaut sauf contrainte projet.
> Pondération = nombre de vidéos sources distinctes : ⭐ (3+), ◐ (2), ○ (1). ⚠ = débat/contradiction.

## Règles

- **⭐ R1 — Livre un logo responsive en variantes officielles, pas un seul fichier redimensionné.** Pourquoi: un logo complet devient illisible en favicon 16px ou en avatar carré. ✓ Produire au minimum : version complète (logotype + picto), réduite (picto/monogramme), micro (initiale pour favicon/avatar), plus monochrome ; afficher moins d'éléments à mesure que l'espace diminue ; prévoir une variante courte/2 lignes pour les emplacements contraints. ✗ Réutiliser le logo complet partout ; déformer/écraser un logo pour le faire rentrer. (src: I_GoPYKdq3k, awdZItdAz7E, 40Tzo3D5UX0 etc.)

- **⭐ R2 — Valide tout logo en monochrome pur AVANT la couleur, et sécurise les formes par des vides, pas par la couleur.** Pourquoi: la version N&B révèle si la forme porte l'identité seule (négatif, gravure, broderie, favicon). ✓ Tester en N&B dès les premières itérations ; livrer mono noir + mono blanc ; séparer les formes par des espaces négatifs ; révéler la couleur seulement une fois la forme validée. ✗ Valider un logo sur sa seule version couleur ; masquer une forme faible derrière la couleur. (src: jhTDQqI7bIM, IS7qnx_YbBE, z4MaPOCNaXU etc.)

- **⭐ R3 — Présente et juge le logo dans son écosystème d'usage réel, jamais sur fond blanc isolé.** Pourquoi: le logo ne pèse qu'1% d'un branding ; un logo qui marche à l'écran peut échouer sur un textile ou un objet 3D. ✓ Présenter des mockups en situation (icône d'app, fond sombre, photo, produit, maillot, social, print) AVANT validation ; juger sur la déclinabilité, pas sur le picto isolé. ✗ Juger un logo seul sur fond blanc comme une identité aboutie. (src: kah5ywUuYAQ, mnn2LFZSpXM, z4MaPOCNaXU etc.)

- **⭐ R4 — Construis un système graphique 360° autour du logo, avec un élément signature systématisé dans toute la DA.** Pourquoi: un logo seul est un symbole orphelin ; c'est la cohérence des assets (typo, couleur, icônes, photo, motion, patterns) qui crée l'impression premium. Répliquer la forme/le rythme central du logo en patterns et fonds rend la marque reconnaissable sans elle ; l'espace négatif est un levier de sophistication. ✓ Décliner le logo en règles (espacements, fonds, typos, set d'icônes même stroke, charte photo, tokens, motion) ; extraire la forme/rythme central et le décliner comme système réutilisable ; documenter chaque touchpoint ; viser 2-3 marqueurs identitaires forts. ✗ Coller le logo en haut à gauche et traiter le reste en générique ; estampiller le motif sur chaque lettre (parcimonie). (src: X3__7JkTEYE, 0F4rWk2MDR8, H0aZGKzdQas, 7LG3jL_7NZY etc.)

- **⭐ R5 — Investis dans une typo propriétaire/caractéristique ; couleur + typo doivent identifier la marque SANS le logo et dériver de son ADN.** Pourquoi: sur une photo ou un post le logotype est souvent absent — fonte et couleur évoquent la marque instantanément ; une fonte gratuite répandue est reproductible par tout concurrent. ✓ Commander une fonte sur mesure (ou variation exclusive d'une open source) déclinée titres/corps/UI ; définir une palette signature qui fonctionne sans le logo ; dériver typo et formes de l'esprit du logo. ✗ Bâtir l'identité sur une fonte système/générique (Arial, Helvetica, Impact) ; choisir typo/formes déconnectées de l'identité. (src: I_GoPYKdq3k, W24Dq6WArL0, gDG_em0qtC0 etc.)

- **⭐ R6 — Assure une cohérence formelle stricte entre pictogramme, typographie et composants UI.** Pourquoi: un picto en courbes douces couplé à une typo anguleuse crée une dissonance perçue comme amateurisme ; un glyph moderne sur une fonte ancienne sonne faux après refresh. ✓ Aligner rayons de courbure, graisses de trait et niveau de régularité sur tout le système ; harmoniser glyph ↔ fonte après refresh ; limiter le nombre de rayons distincts. ✗ Mélanger formes géométriques pures et tracés à main levée, ou cumuler des rayons disparates sans raison. (src: I_GoPYKdq3k, bATtbFNqbl4, ZKU6vLJN18U etc.)

- **⭐ R7 — Conçois pictogramme et logotype pour qu'ils fonctionnent CHACUN seul ; système modulaire combinable.** Pourquoi: un système mature affiche le nom seul, le symbole seul ou les deux selon le support ; un lock-up monolithique casse dès qu'on change d'échelle. ✓ Tester le picto sans le nom (icône app, favicon) et le nom sans le picto (header, email), les deux identifiables ; définir des composants combinables lisibles du tampon au grand format. ✗ Fusionner nom et symbole au point qu'ils ne s'utilisent pas séparément ; figer un seul lockup rigide. (src: DLCU5CJnqLg, jhTDQqI7bIM, H0aZGKzdQas etc.)

- **⭐ R8 — Préserve l'élément distinctif qui rend la marque reconnaissable ; vise une signature spécifique à son secteur, pas une exécution juste « propre ».** Pourquoi: quand tous les logos d'un secteur convergent vers le même flat épuré, la simplicité cesse de différencier ; un logo trop générique pourrait se poser sur « un steakhouse ou des fournitures de bureau ». ✓ Chercher l'asset qui identifie la marque sans le nom et le sublimer ; ancrer un signe qui dit le métier ; confronter aux marques connues du registre pour écarter toute ressemblance. ✗ Suivre le « blending » minimaliste du secteur ; choisir le bleu « parce que digital » ; reprendre la palette/forme de lettres d'un leader → confusion. (src: Y4cY0i1w6NM, le77ZiMUST8, 5scZtI4rwBA etc.)

- **⭐ R9 — Conçois d'abord en silhouette : un logo doit pouvoir être redessiné de mémoire ; la lisibilité (petite taille ET distance) est la condition de la mémorisation.** Pourquoi: si l'œil ne déchiffre pas, le cerveau n'encode pas ; un logo vit le plus souvent en miniature (favicon, onglet). La perfection = retirer, pas ajouter. ⚠ Nuance: pour une marque déjà iconique, la visibilité distinctive peut primer sur le déchiffrage littéral (un picto réussi se « devine ») ; une marque inconnue a besoin de lisibilité. ✓ Concevoir en silhouette (noir uni, à distance) avant détail/couleur ; tester à 16-24px ; faire reproduire de mémoire par des non-designers. ✗ Empiler formes/détails/dégradés/3D qui s'effondrent en petit ; exiger une légende pour qu'un logo soit compris. (src: ZjVMbHfeFFA, DPa_SicGS9Q, 5scZtI4rwBA, DLCU5CJnqLg etc.)

- **⭐ R10 — La perception de qualité (finition, vectorisation) prime sur la performance réelle pour la confiance ; aligne le soin sur la promesse, le prix et la réalité produit.** Pourquoi: un produit bien designé inspire confiance ; mais une belle interface crée une promesse que la livraison doit tenir ; un logo pixelisé/raster plombe toute la marque. ✓ Investir dans la qualité visuelle et la cohérence ; aligner le soin sur le positionnement ; fournir le logo en vectoriel net (SVG/PDF) à toute taille. ✗ Vendre des teasers léchés puis livrer une UI bâclée ; afficher une interface premium pour un produit bas de gamme ; ship un logo flou/raster sous-dimensionné. (src: Z5TjBuYUmNI, DrY4WPlIFnQ, pNC_oKWlgy8 etc.)

- **⭐ R11 — Pars d'un concept fort et d'un brief structuré (qui, pour qui, intention) AVANT la technique ; UN angle bien exécuté, pas une accumulation.** Pourquoi: sans idée claire, empiler effets/détails ne fait que combler le vide ; une texture qui « fait pro » masque souvent un concept faible. Empiler blason + médaille + étoile + ombres + dégradés dilue le message — « moins d'effets, c'est mieux ». ✓ Demander but/audience/intention avant d'évaluer ; extraire 2-3 mots-clés et ne logoter que ceux-là ; simplifier tout picto à ses formes essentielles (s'il ne tient pas en 5-7 formes simples, il est trop complexe). ✗ Foncer dans la technique sans concept ; compter sur une texture pour sauver un logo sans idée ; cumuler ombres + dégradés + lumières qui brouillent forme et sens. (src: 5scZtI4rwBA, Pzd028fwX8o, uMjoqUwxltU etc.)

- **◐ R12 — Maintiens une cohérence systémique dans une famille de marques : design system partagé d'abord, identités ensuite ; marque parapluie neutre.** Pourquoi: sans dénominateur commun (police, casse, espacement), l'écosystème ressemble à un « randomizer » ; un sous-produit gagne à être une déclinaison reconnaissable du parent. ✓ Définir les règles communes (fonte, casse, grammaire d'arrondis/couleur) avant chaque identité ; réutiliser formes/lettres du logo principal ; garder le mark groupe neutre (façon Alphabet). ⚠ Débat: pour des publics réellement distincts, une identité propre (reliée par un code discret) peut créer plus de valeur que l'homogénéisation forcée — trancher selon l'écart entre cibles. ✗ Traiter chaque logo comme un projet indépendant ; baker la saveur d'un produit dans le mark groupe. (src: bATtbFNqbl4, 2ml0fAuIQTo, njcLhEd3rd0 etc.)

- **◐ R13 — Distingue refresh et redesign : sur une marque établie, conserve un fil conducteur avec l'asset le plus fort plutôt que tout effacer.** Pourquoi: l'équité de marque est dans les éléments reconnaissables ; la rupture totale crée un sentiment de trahison. Moderniser = retirer l'ornement daté (boîtes grises, faux 3D) vers des formes plus plates. ✓ Identifier l'asset sémantique le plus fort et l'encoder de façon contemporaine ; glisser un clin d'œil discret à l'ancien ; rafraîchir l'écosystème autour. ⚠ Débat: si le pivot de cible est réel et total, assumer la rupture vaut mieux qu'un compromis ambigu. ✗ Repartir d'une page blanche pour une marque iconique sans pont visuel ; garder dégradés/pseudo-3D qui ne signalent que l'âge. (src: jhTDQqI7bIM, iXpoUKPJ19I, H0aZGKzdQas etc.)

- **◐ R14 — Adopte une identité chromatique forte, hiérarchisée et cohérente avec l'univers sémantique ; la couleur est souvent l'identifiant le plus puissant.** Pourquoi: la couleur déclenche des associations sectorielles inconscientes (un vert mal choisi « fait poulet bio ») ; au-delà de 2-3 couleurs sans hiérarchie, le logo « brûle les yeux ». ✓ Choisir une dominante ~80% + un accent ; tester ce qu'évoque la couleur auprès de personnes hors projet ; assigner des rôles précis. Code-couleur par catégorie : réserver à un set fixe restreint (la roue manque de teintes distinctes agréables à l'échelle) — sinon une seule couleur de marque + un autre différenciateur. ✗ Empiler 3+ couleurs sans hiérarchie ; complémentaires pures saturées (vert+rouge) en contact → brûlure oculaire ; s'engager sur « une couleur par catégorie » pour une taxonomie ouverte. (src: 5jA49AXm-H0, 8b1I7Gk-fss, Pzd028fwX8o, njcLhEd3rd0 etc.)

- **◐ R15 — Conçois le logo en motion dès le départ pour le digital ; construction précise, mais n'anime jamais la couleur du texte.** Pourquoi: un logo cross-platform moderne vit en animation (hover, loading, états) ; toute approximation géométrique (un angle qui ne ferme pas) saute aux yeux, surtout si la révélation imite une intro célèbre ; faire boucler un changement de couleur sur le nom nuit à la lecture. ✓ Définir 2-3 états animés ; rendre la construction précise (angles qui ferment) avant d'animer ; limiter toute animation de couleur à un élément graphique isolé. ✗ Livrer un logo uniquement en SVG statique ; ship une animation qui copie nûment une intro connue ; faire boucler la couleur du nom. (src: cY4w3Um2RJI, FK44kBj4elI, 5jA49AXm-H0 etc.)

- **◐ R16 — Donne du dynamisme et une direction si l'univers l'appelle, tout en gardant l'équilibre de masse et la hiérarchie symbole > ornements/texte.** Pourquoi: un logo incliné communique l'énergie mieux qu'un logo plat figé (orientation vers la droite/haut = « avant ») ; mais le centre de gravité perçu affecte la stabilité, et un élément central doit rester la star — tout ornement (doubles contours, texte) qui rivalise dilue le signe. ✓ Expérimenter une légère rotation/courbe intentionnelle quand le sens l'appelle ; tester plusieurs ratios symbole/texte où les deux respirent ; distribuer le blanc autour du centre de masse. ✗ Logo 100% rectiligne figé pour un univers d'énergie ; logo qui « penche » par accident ; texte trop gros qui concurrence le symbole. (src: mM7HmbLxTC4, Pzd028fwX8o, 5jA49AXm-H0, 44L6uJKFg_4 etc.)

- **○ R17 — Construis un set d'icônes cohérent : chaque taille vérifiée individuellement, orientation/construction unique, custom si l'identité est forte.** Pourquoi: redimensionner un seul fichier rend le stroke trop fin/épais ; mélanger icônes de face et en perspective brise l'unité ; des icônes Material/Font Awesome standards cassent l'immersion d'une identité forte. ✓ Ajuster légèrement le stroke par taille (16/20/24px) selon les guidelines plateforme ; appliquer une règle de construction unique (orientation, viewpoint) sans exception ; investir dans des pictos reconnaissables comme appartenant à la marque. ✗ Un seul fichier mis à l'échelle pour toutes les tailles ; icônes de profil et de face mêlées ; icônes génériques dans une interface à identité forte. (src: 0F4rWk2MDR8, c75EZmfFCQ4, CnMMmb7Usus etc.)

- **○ R18 — Différencie visuellement les icônes d'apps d'un même écosystème pour les distinguer sans lire le label.** Pourquoi: une suite dont les icônes partagent forme/couleur/style oblige à lire les noms, génère erreurs de tap et frustration. ✓ Attribuer à chaque app une couleur, une forme ou une orientation distincte, tout en gardant un lien visuel cohérent avec la famille. ✗ Homogénéiser au point que les icônes deviennent interchangeables au premier regard. (src: tPEhs-cdYp0, Z5TjBuYUmNI)

- **○ R19 — Pour badge/certification/étiquette/textile, propose une version conteneur, un ratio proche du carré, des contours bold et stables.** Pourquoi: un symbole libre s'efface sur certains fonds ; un logo vertical très allongé est systématiquement recadré/compressé ; les finesses disparaissent à la couture. ✓ Fournir une version « en pastille/macaron/écusson », viser un ratio entre 1:1 et 3:2 ; simplifier le picto jusqu'à le rendre lisible en broderie ~2cm. ✗ Utiliser le logo libre sans conteneur ; créer un logo 2× plus haut que large ; garder des lignes < 2px sur un logo textile. (src: 2S9ScSXzBNM, kah5ywUuYAQ, E8MM2MdUrw4 etc.)

- **○ R20 — Garde picto et mot séparés et lisibles : ne remplace pas une lettre par une icône, n'intègre pas de texte dans un picto, sépare les ligatures involontaires.** Pourquoi: remplacer une lettre par une image rompt la lecture (« mot + image ») ; un texte fondu dans un picto devient illisible sous 32px ; une ligature comprime l'espace intérieur. ✓ Garder picto et mot complémentaires avec règles responsive ; vérifier les paires de lettres qui se touchent et les séparer. ✗ Insérer un picto à la place d'un caractère ; fondre des abréviations/chiffres dans un picto d'app ; conserver une ligature qui nuit à la lisibilité. (src: -gkGGmd8x_8, tPEhs-cdYp0, Z5TjBuYUmNI etc.)

- **○ R21 — Conçois pour la démographie cible réelle (et le problème résolu), pas pour les designers ; aligne le registre sur l'émotion du produit.** Pourquoi: la cible détermine le langage graphique — un logo grand public ne doit pas être optimisé pour les réactions de designers ; un produit chaleureux habillé en froid/corporate trahit sa promesse. ✓ Identifier à qui le logo parle en usage réel ; relier la forme au bénéfice central ; respecter le brief stylistique validé ; choisir couleurs/formes qui traduisent le bon registre ; cibler les références culturelles sur l'audience. ✗ Optimiser pour les réactions de designers ; ajouter des effets quand le client a validé sobre ; habiller un produit chaleureux en « froid parce que c'est tech ». (src: tLf9cPzF3u8, yV4PUOHzuJk, 5scZtI4rwBA etc.)

- **○ R22 — Vérifie la cohérence nom ↔ identité visuelle et tiens un brand kit propre et délégué.** Pourquoi: un nom plus « fancy » fait paraître l'offre premium, un handle cryptique (S4W6) nuit à la mémorisation ; un nom évoquant une matière (« noir nacré ») affiché en bleu crée une dissonance ; une fonte off-brand résiduelle fragmente la voix de marque. ✓ Choisir un nom lisible/prononçable cohérent avec le positionnement ; matcher la palette dominante avec ce qu'il évoque (distinguer couleur du logo vs charte) ; normaliser le kit aux typefaces choisies, garder les assets composables éditables, sourcer depuis les canaux officiels ; commander une mini-charte à un designer si le branding n'est pas son métier. ✗ Garder un nom cheap pour monter en gamme ; aplatir un asset composable en PNG fixe ; récupérer un logo via Google Images/Pinterest. (src: R1DNknJvp64, IVFuk2eP77I, Iq8wqN2upmU, Awoo_YRhaS8 etc.)

## Anti-patterns

- ✗ Réutiliser le logo complet en favicon/avatar/app icon → bouillie illisible ; déformer/écraser pour le faire rentrer. (R1, R19)
- ✗ Valider un logo sur sa seule couleur sans tester N&B ni 16px ; sécuriser les formes par la couleur. (R2)
- ✗ Juger un logo sur fond blanc isolé, sans le voir vivre en motion/app/print/social/textile. (R3)
- ✗ Changer le logo en gardant typo/photo inchangées → symbole orphelin ; livrer un assemblage neutre sans 2-3 marqueurs forts ; estampiller le motif sur chaque lettre. (R4)
- ✗ Bâtir l'identité sur une fonte système/générique ; typo/formes déconnectées de l'ADN du logo. (R5)
- ✗ Coupler picto rond/doux à typo anguleuse ; mélanger géométrique pur et main levée ; rayons disparates sans raison. (R6)
- ✗ Logo qui n'existe qu'en picto+nom inséparable, ou lockup unique rigide. (R7)
- ✗ Suivre le « blending » du secteur ; bleu « parce que digital » ; reprendre la palette/lettres d'un géant → confusion. (R8)
- ✗ Logo qu'un non-designer ne peut redessiner de mémoire ; traits fins illisibles en petit ; exiger une légende. (R9)
- ✗ Teasers léchés puis UI bâclée ; interface premium pour produit bas de gamme ; logo pixelisé/raster. (R10)
- ✗ Foncer dans la technique sans concept ; texture-cache-misère ; cumuler ombres + dégradés + lumières ; picto > 5-7 formes. (R11)
- ✗ Sous-marques choisissant chacune police/casse/couleur (« randomizer ») ; saveur produit bakée dans un mark groupe neutre. (R12)
- ✗ Page blanche pour une marque iconique sans pont visuel ; garder dégradés/pseudo-3D datés. (R13)
- ✗ 3+ couleurs sans hiérarchie ; complémentaires pures saturées en contact ; « une couleur par catégorie » sur taxonomie ouverte ; couleur hors-sujet sémantiquement. (R14)
- ✗ Logo uniquement en SVG statique ; animation copiant une intro connue ou aux angles ouverts ; couleur du nom animée. (R15)
- ✗ Logo rectiligne figé pour un univers d'énergie ; penchant accidentel ; texte/ornement qui concurrence le symbole. (R16)
- ✗ Un seul fichier d'icône mis à l'échelle ; icônes de face et en perspective mêlées ; icônes génériques sur identité forte. (R17)
- ✗ Icônes quasi identiques pour plusieurs apps d'une suite → erreurs d'ouverture. (R18)
- ✗ Logo vertical ≥ 2× sa largeur, éléments qui dépassent du bloc, lignes < 2px en textile, symbole libre sans conteneur. (R19)
- ✗ Remplacer une lettre par une icône, fondre du texte dans un picto, conserver une ligature involontaire. (R20)
- ✗ Optimiser pour les réactions de designers ; habiller un produit chaleureux en froid/corporate. (R21)
- ✗ Nom cheap pour monter en gamme ; handle cryptique ; palette qui contredit le nom ; fonte parasite ; logo via Google Images. (R22)

## Checklist build

- [ ] Variantes officielles : complète, réduite (picto), micro (favicon), monochrome noir + blanc ; variante courte pour emplacements contraints (R1)
- [ ] Logo validé en silhouette N&B pur ET à 16-24px AVANT la couleur ; formes sécurisées par des vides ; redessinable de mémoire (R2, R9)
- [ ] Picto seul et logotype seul fonctionnent chacun ; système modulaire combinable ; lettres/texte non fondus dans le picto, ligatures séparées (R7, R20)
- [ ] Cohérence formelle picto ↔ typo ↔ UI (rayons, graisses, terminaisons) ; glyph ↔ fonte après refresh (R6)
- [ ] Système 360° : typo, palette, iconographie (même stroke), charte photo, patterns, motion, touchpoints ; élément signature systématisé avec parcimonie ; 2-3 marqueurs forts (R4)
- [ ] Couleur + typo identifient la marque sans le logo ; fonte propriétaire/caractéristique dérivée de l'ADN, pas générique (R5)
- [ ] Logo présenté en contexte réel (app, fond sombre, photo, produit, textile, motion), jugé sur sa déclinabilité ; signature spécifique au secteur, sans confusion avec un géant (R3, R8)
- [ ] Qualité visuelle alignée sur la promesse/le prix ; logo en vectoriel net (SVG/PDF) (R10)
- [ ] Concept fort + brief (qui/pour qui/intention) AVANT la technique ; UN angle, picto en 5-7 formes max, pas d'effets cumulés (R11)
- [ ] Palette limitée : dominante ~80% + accent ; pas de complémentaires pures en contact ; code-couleur par catégorie réservé à un set fixe (R14)
- [ ] Ratio du bloc entre 1:1 et 3:2 ; version pastille + contours bold pour badge/certification/textile (R19)
- [ ] 2-3 états animés pour le digital ; angles qui ferment ; aucune animation de couleur sur le nom ; pas de copie d'intro (R15)
- [ ] Dynamisme/direction si l'univers l'appelle ; pas de penchant accidentel ; équilibre de masse, hiérarchie symbole > ornements (R16)
- [ ] Set d'icônes : chaque taille vérifiée (guidelines plateforme), orientation unique, custom si identité forte ; apps d'un écosystème distinctes (R17, R18)
- [ ] Logo conçu pour la démographie cible réelle, registre aligné sur l'émotion du produit, brief client respecté (R21)
- [ ] Nom lisible/prononçable cohérent avec le positionnement ; palette alignée sur ce qu'il évoque ; brand kit propre, assets composables éditables, sourcés officiellement (R22)
- [ ] Famille de sous-marques : design system commun avant les identités ; mark groupe neutre. Refresh vs redesign tranché ; fil conducteur réinterprété (R12, R13)
